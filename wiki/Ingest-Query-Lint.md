# Ingest-Query-Lint

## 概要
[[LLM Wiki]] において AI が担う3つの基本操作。[[3層構造]] の raw / wiki / schema を回すための入力・出力・保守のサイクルを構成する。

## 本文

### Ingest（取り込み）
- raw 層に投入された新しい資料を読み込み、wiki に反映する操作。
- 概念を抽出し、関連する既存ページ（カーパシー氏の事例では10〜15ページ程度）を自動更新する。
- 新規概念があれば新しいページを作成し、既存ページとの相互リンクを張る。
- 本リポジトリでは `/ingest` スラッシュコマンドで起動。

### Query（質問と書き戻し）
- ユーザーが wiki に対して質問する操作。
- 単に回答するだけでなく、有用な回答は **wiki に書き戻して資産化** できる点が特徴。
- これにより会話で得られた知見が消費されずに蓄積される。
- 本リポジトリでは `/query` スラッシュコマンドで起動。

### Lint（整合性検査）
- wiki 内部の矛盾・重複・孤立ページ・dead source などを検出する操作。
- raw にあるのに wiki に反映されていない処理漏れも検知できる。
- 自動で修正するのではなく **検出して人間に報告** することが多い（誤修正を避けるため）。
- 本リポジトリでは `/lint` スラッシュコマンドで起動。

### 3操作の関係
- Ingest が「入力」、Query が「出力かつ追加入力」、Lint が「保守」と整理できる。
- どれもAI単独で完結させず、人間がレビューできる粒度で動かすのが推奨される。

## 入力プロンプト例（原典より）

原典（note 記事）では、Cursor のチャットパネルに以下のような自然文を入力して各操作を起動する運用が紹介されている。本リポジトリではこれをスラッシュコマンドに移植している。

```
raw/ に新しく追加された記事を読んで、wiki/ の該当ページに情報を追記してください
```

```
raw/ のファイルを読み込んで、概念ごとに wiki/ のページへ整理してください
```

```
wiki を参照して、過去のSNS広告施策で効果が高かったパターンを教えて
```
