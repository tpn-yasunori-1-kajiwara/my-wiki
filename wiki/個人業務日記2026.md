# 個人業務日記2026

## 概要

著者本人の 2026 年度の業務メモ。出来事ベースの時系列ログで、後から振り返るための原資料。テーマごとの掘り下げは個別ページ（下記「関連」）に分割し、本ページは **インデックス兼一次ログ** として保つ。

## 本文

### 3月（年度末）

- **03/23** 社長表彰の申請書作成に苦労。社用携帯が初期化される事故。
- **03/24** タスクが少なくテレワーク。部下面談・考課を実施。
- **03/25** 小笠原さんと意見交換。熱量がすごい。
- **03/26** **Agent Garden** に Agent が追加されているのを確認。津野君と Agent デプロイを試す予定。
- **03/27** 同じ AI 機能でも、社内開発なら 2 日、外注なら 1 ヶ月かかる。差はコスト（金・時間・セキュリティ・要件）。「**仲間と思われないとダメ**」。
- **03/30** [[AI Mart]] の今後の開発方針議論。特定分野で高機能化／単機能を集約して大機能化／**使えそうなら即作って即出す（使われなければ消す）の路線維持**、の3案。著者は3つ目を残したい立場。
- **03/31** マーケDX事業部の幹部会で来季計画発表。**営業改革** が重点テーマ。営業履歴・取引履歴・提案書・ノウハウを RAG でインプットし「勝てる提案」を出力する AI システム構想。著者の対案は [[営業改革 (マーケDX事業部)]] 参照。

### 4月

- **04/01** 久々に開発。**Mermaid 可視化ツール**に LLM コード自動生成とインフォグラフィックス生成機能を追加。Codex 利用で 3〜4 時間（旧来なら 2〜3 日）。詳細は [[Mermaid可視化ツール]]。
- **04/02** AI ビジネスは難しい。社内効率化・AI リテラシー向上は成果を出せたが、外販ビジネス化となると何をサービスにすればいいか不明。**内製化の風潮** と相まって DX 分野で事業会社以外の仕事はなくなる予感。→ [[AI企業の受託シフト]] と同方向の所感。
- **04/03** 社内政治と予算の取り合いがめんどくさい。AI のおかげで週の仕事は 4 日で完了できている。
- **04/06** 終日 MTG。大井田くんの録音／メモのデータ化アプリと、クラウド上 CLI 開発アプリに期待。導入ハードルはあるが「劇薬的に投入する」方向。
- **04/07** 今週は **Skills 開発** をしたい。HTML⇔PPTX の Skill を作りたいが実現可能か未知。
- **04/08** 会社 PC を交換し終日セットアップ。
- **04/09** **Codex が認証エラー**（PC 交換の影響か）。担当窓口に問合せ中、GeminiCLI で代替。
- **04/10** Claude Code を業務で使っている人の話を聞いた。便利そうだが申請単位・支払主体がボトルネックになりそう。
- **04/13** まじんさんの **Canvus サイドバー拡張** を真似て、**Gem 呼び出し Chrome 拡張** を作ることに決めた。詳細は [[Chrome拡張開発]]。
- **04/14** CSM 方針発表。営業改革の具体策として、**営業に Claude Code を持たせてデモ・プロトタイプ開発／顧客コミュニケーションを AI 支援** する方向を提案中。開発中のクラウド上 Claude アプリをフックに、最終的に個別 PC 配布へ。
- **04/15** チームの黎明期は過ぎたので、今年度でお役御免かなと感じる。**維持・発展は得意ではない**。今年度中に [[AI Mart]] 全社展開と業務効率化のビジネス化を達成できれば一区切り。
- **04/16** **Chrome Skills** が発表。Claude／Codex とは思想が違うと感じた。Claude/Codex は **開発の効率化・拡張**、Chrome Skills は **娯楽・コミュニケーション含めたブラウザ拡張**。→ [[Chrome Skills]]
- **04/17** **TODO 登録 Chrome 拡張**を開発することに決定。自分が使って便利か判断する。
- **04/20** Skills を初めて作成。**調整・チューニングに時間がかかる**ので面倒。
- **04/21** 急に忙しくなってきた。案件相談が増加。開発もできておりバランスは良い。
- **04/22** ブラマケの提案情報収集効率化を **Codex + Skills** で行うことに決定（担当者の AI リテラシー向上も狙い）。TODO 拡張は要望（予定化・日時指定）に対応すると右クリック後の入力画面が必要になり quick 登録性を失うため、**現状のままリリース**。
- **04/23** **東芝の件**の金額設計が難しかった。**削減効果の 35% を従量課金**でもらう契約に。詳細は [[東芝従量課金案件]]。
- **04/24** **全社 AI 推進室のグローバル法改正情報発信高度化** 案件。資料読込・整理の効率化は可能、Web から信頼性高く情報収集する部分は難しい。期待値コントロールが鍵。
- **04/27** 風邪でテレワーク・エコモード。
- **04/28** GW 前最終日。風邪未治癒。
