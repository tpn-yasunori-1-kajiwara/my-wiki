# LLM Wiki

## 概要
AI研究者アンドレイ・カーパシー氏が提唱した、LLMと併走させながら育てる「永続的で相互リンクされた個人用ウィキ」のアプローチ。会話ログのように消費されて消えていく知識を、整理された資産へ変換することを目的とする。

## 本文

### 従来のChatGPT利用との違い
- ChatGPTとの対話は一回限りの「消費」になりがちで、得た知識が次の会話に蓄積されない。
- LLM Wikiは知識を概念ごとに記事化し、相互リンクして「育てる」ことで、過去の知見を再利用可能な資産にする。
- RAG（その場で関連文書を検索して回答）と異なり、AIが**事前に**ウィキを編纂しておく点が特徴。

### 何で構成されるか
- **3層の論理構造**（[[3層構造]] を参照）: raw / wiki / schema の3レイヤーに役割を分離する。
- **AI側の3つの操作**（[[Ingest-Query-Lint]] を参照）: Ingest（取り込み）・Query（質問と書き戻し）・Lint（整合性検査）。

### 実例
- カーパシー氏自身は約100記事・40万語規模の知識ベースを構築している。
- 公開されている類例として [[FarzaPedia]] がある。

### 利用者の操作の簡素化
著者（森本康人氏）は「AIをコード操作の道具から知識整理の共同編集者へ転換する」と表現している。利用者がやることは原則：

1. 素材を raw 層に投入する
2. Cursor/Claude Code 等のチャットで指示を出す

の2つに集約される。

### 限界と課題
- AI生成コンテンツの誤りが混入するリスク。
- 大規模化したときに検索基盤（インデックス・全文検索）が必要になる。
- schema 設計（粒度・リンク方針・テンプレート）を最初に詰めておかないと後で破綻しやすい。
