# /ingest — 図解PNGをwikiに取り込む

CLAUDE.md のルールに厳密に従って、`raw/` の図解を `wiki/` に反映してください。

## 引数の解釈

- 引数が空、または `all`: `raw/` 内の **未処理** のPNGすべてを対象。
- 引数がファイル名（または複数のファイル名）: そのファイルのみを対象。

「未処理」の定義: `wiki/` のどのページの `sources:` フロントマターにも、そのPNGのファイル名が記録されていない状態。

## 手順

1. **対象PNGの列挙**
   - `raw/` 配下の `*.png` を Glob で取得。
   - 各PNGについて、Grep で `wiki/**/*.md` を検索し `sources:` に記載されているかを確認。
   - 未処理のものだけリスト化。引数で明示指定された場合はそれを優先。

2. **各PNGを処理**（1枚ずつ順番に）
   - Read ツールでPNGを **画像として** 読む。視覚情報から内容を理解すること。**ファイル名から推測しない**。
   - PNGに含まれる主要な概念を列挙する（3〜10個目安）。

3. **既存ページとの照合**
   - 各概念について、`wiki/` のファイル名・title・aliases を確認。
   - Grep で全文検索もし、類似概念がすでに別名で存在していないか確認。

4. **編集**
   - 既存ページがあれば: Edit で本文に追記、`sources:` にファイル名追加、`updated:` を `$(今日の日付)` に更新。
   - 既存ページがなければ: Write で新規作成。CLAUDE.md のフロントマターテンプレートに従う。
   - **相互リンク**: 新規・更新ページから関連する既存ページへ `[[ページ名]]` を張る。逆方向の追記も行う。
   - **矛盾検出**: 既存記述と新出典が矛盾する場合は、勝手に判断せず該当箇所に `> **要レビュー:** ...` フラグを残す。

5. **進捗報告**
   - 処理した各PNGについて、「新規作成」「更新」したページ名と件数を最後にまとめて報告する。
   - 例:
     ```
     2026-05-12_data-pipeline.png
       新規: データパイプライン.md, バッチ処理.md
       更新: ETL.md（出典追加・本文に1段落追加）
     ```

## 重要

- `raw/` のファイルを **絶対に削除・改名・編集しない**。
- 画像を見ずにファイル名やフォルダ名だけで内容を断定しない。
- フロントマターは省略しない。
- 不明点は本文に「不明: 〜」と明示する。
- 1枚あたり数ページ生成しても構わない（CLAUDE.md の粒度ルール参照）。
