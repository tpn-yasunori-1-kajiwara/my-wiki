# /ingest — テキスト資料をwikiに取り込む

CLAUDE.md のルールに厳密に従い、`raw/` の資料を `wiki/` に反映してください。
**raw/ のファイルは削除も移動もしません**。処理済み判定は `wiki/*.md` の `sources:` で行います。

## 引数の解釈

- 引数が空、または `all`: `raw/` 内のすべての未処理ファイルを対象。
- 引数がファイル名: そのファイルのみを対象（処理済みでも引数指定があれば再処理）。

「未処理」の定義: `wiki/` のどのページの `sources:` にも、そのファイル名が記載されていない状態。

## 手順

1. **対象列挙**
   - Glob で `raw/*` を取得（`.gitkeep` は除外）。
   - 各ファイル名について Grep で `wiki/**/*.md` の `sources:` 出現を確認。
   - 未処理のものだけリスト化（引数指定時はそれを優先）。

2. **各ファイル処理**（1件ずつ）
   - Read ツールでテキスト/PDF/HTML を読む。

3. **URL抽出と取得**
   - 本文中の URL（`https?://...`）を列挙。
   - 各 URL を **WebFetch** で取得し、内容を把握（要約・引用に使える形で）。
   - 取得失敗・認証必要なURLはスキップし、wikiページ本文に「不明: URL取得失敗（理由）」と注記。

4. **概念抽出**
   - テキスト本体 + 取得した URL 内容から主要な概念を3〜10個列挙。

5. **既存ページとの照合**
   - 各概念について Grep で `wiki/**/*.md` を検索。
   - title / aliases / ファイル名で照合。

6. **編集**
   - 既存ページあり → Edit で追記、`sources:` にファイル名追加、`updated:` を今日の日付に更新、必要に応じて `references:` にURL追加。
   - なし → Write で新規作成。CLAUDE.md のフロントマターテンプレートに従う。
   - **画像埋め込みは使わない**（テキストのみ）。
   - 相互リンクを張る（`[[ページ名]]`）。
   - 矛盾検出時は `> **要レビュー:** ...` フラグを残す。

7. **進捗報告**
   - 各ファイルについて、新規作成/更新したページと、参照したURL数をまとめて報告。
   - 例:
     ```
     2026-05-12_note.md
       新規: 個人用Wiki.md
       更新: Claude-Codeの業務活用.md（出典追加）
       URL参照: 1件（note.com/...）
     ```

## 重要

- **raw/ のファイルを絶対に削除・改名・編集しない**。
- 画像埋め込み (`![](...)`) を使わない。
- 推測で書かない。不明点は「不明: 〜」と明示する。
- URL取得は適度な範囲で（無限再帰や明らかに大量になる場合は数件で切り上げる）。
- フロントマターは省略しない。
